# STM32F103 简易示波器与逻辑分析仪

这是一个基于 STM32F103C8T6 和 0.96 寸 SSD1306 OLED 的简易信号观察工具，支持模拟示波器模式和四通道逻辑分析仪模式。工程使用 STM32 标准外设库，适合学习 ADC、DMA、定时器采样、GPIO 采样、OLED 显示和按键交互。

## 功能

- 模拟示波器模式：显示 ADC 采样波形、最大电压 MAX、峰峰值 VPP。
- 逻辑分析仪模式：支持 CH0~CH3 四通道数字信号显示。
- TIM2 + DMA 逻辑采样：TIM2 产生固定采样节拍，DMA 自动读取 GPIOA->IDR，降低 CPU 中断负担。
- 空闲后边沿触发：DMA 采集一块数据后查找“空闲一段时间后的第一个边沿”，更容易抓到通信开始位置。
- 采集后自动定格：方便观察 I2C、SPI、UART、PWM 等数字波形。
- 左右查看波形：OLED 一屏显示不完整时，可以移动显示窗口。
- 按键切换模式：不需要重新烧录程序即可切换示波器和逻辑分析仪。

## 硬件连接

| 功能 | STM32 引脚 | 说明 |
| --- | --- | --- |
| 示波器输入 | PA1 | ADC1_CH1，输入范围建议 0~3.3V |
| 逻辑 CH0 | PA0 | 数字信号输入 |
| 逻辑 CH1 | PA1 | 数字信号输入；与示波器输入共用引脚，使用时注意模式 |
| 逻辑 CH2 | PA2 | 数字信号输入 |
| 逻辑 CH3 | PA3 | 数字信号输入 |
| 模式切换按键 | PC14 | 切换示波器/逻辑分析仪，按键另一端接 GND |
| 协议切换按键 | PA4 | 在 IIC/SPI/UART 触发模式间切换，按键另一端接 GND |
| 左移按键 | PB0 | 逻辑分析仪画面左移，按键另一端接 GND |
| 右移按键 | PA6 | 逻辑分析仪画面右移，按键另一端接 GND |
| 下一帧按键 | PB10 | 采集下一帧，按键另一端接 GND |

被测板和分析仪板必须共地：

```text
被测板 GND  ----  分析仪 GND
被测信号线  ----  对应输入引脚
```

## 使用方法

1. 烧录程序后，默认进入 `SCOPE` 示波器模式。
2. 按下 `PC14` 按键，在 `SCOPE` 和 `LOGIC` 模式之间切换。
3. 示波器模式下，将 0~3.3V 模拟信号接入 `PA1`。
4. 逻辑分析仪模式下，将数字信号接入 `PA0~PA3`。
5. 逻辑分析仪采集到边沿后会自动定格。
6. 定格后：
   - `PA4`：切换 IIC/SPI/UART 触发模式，OLED 顶部会显示当前模式。
   - `PB0`：向左查看更早的采样点。
   - `PA6`：向右查看更晚的采样点。
   - `PB10`：采集下一帧。

## PWM 观察说明

PWM 的占空比不是电压高度。对于 3.3V PWM：

- 0% 占空比：一直是 0V，MAX=0，VPP=0。
- 50% 占空比：0V 和 3.3V 各占一半时间，MAX 约 3300mV，VPP 约 3300mV。
- 100% 占空比：一直是 3.3V，MAX 约 3300mV，VPP=0。

如果想直接看到接近 1.65V 的模拟电压，需要给 PWM 后面加 RC 滤波。

当前逻辑分析仪默认使用“空闲后边沿触发”，更适合 I2C、SPI、UART 这类一段一段出现的通信。测连续 PWM 时，如果一直等待不到触发，可以把 `driver/Logic.h` 中的 `LOGIC_IDLE_BEFORE_TRIGGER_SAMPLES` 改为 `0`，恢复普通边沿触发。

## 工程结构

```text
app/
  main.c      程序入口，只保留主循环
  App.c       应用层逻辑：模式切换、示波器显示、逻辑分析仪流程
  App.h       应用层接口
driver/
  AD.c/h      ADC + DMA 采样
  Timer.c/h   示波器采样定时器
  Logic.c/h   逻辑分析仪采样，使用 TIM2 + DMA1_Channel2 读取 GPIOA->IDR
  Key.c/h     按键检测
dsp/
  OLED.c/h    OLED 显示驱动和波形绘制
System/
  Delay.c/h   延时函数
```

## 编译环境

- MCU：STM32F103C8T6
- 开发环境：Keil uVision5
- 编译器：ARMCC 5
- 固件库：STM32F10x Standard Peripheral Library

打开 `project.uvprojx` 后直接编译即可。

## 注意事项

- 输入电压不要超过 3.3V，避免损坏 STM32 IO 或 ADC。
- 测量外部板子的信号时必须共地。
- 该项目用于学习和低速信号观察，不能替代专业示波器或专业逻辑分析仪。
- OLED 分辨率有限，高速信号会被压缩显示；当前 OLED 主要作为辅助显示，后续可以把 DMA 采样数据发给 PC 上位机。
